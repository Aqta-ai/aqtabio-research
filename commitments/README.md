# Public commitment ledger

This folder is an append-only public record of forward-looking risk
predictions made by AqtaBio. Each Monday a new file
`commitments/YYYY-WWW.json` lands here, signed by the founder's
GPG/SSH-verified git commit. Each file names the highest-risk seeded
tiles per pathogen for the week ahead, **before any outbreak in that
window has been notified by WHO, ECDC, or a national MoH**.

The intent is to convert AqtaBio's claim from "we caught these
historical events in backtest" to "we are placing public, dated,
independently-verifiable bets and being measured against outbreaks
we did not pick." Hits and misses are both visible. Cherry-picking
is impossible because the commitment exists in the public git
history before the outbreak it claims to anticipate.

Each commitment is fixed at the moment of the commit; subsequent
edits to the file would not survive a `git log --follow` audit.

## How to verify

For any outbreak notified by WHO Disease Outbreak News, ECDC weekly
bulletin, or national MoH:

1. Note the affected location and the public-notification date.
2. Identify the AqtaBio 25 km tile that contains the centroid of
   the affected location. The tile-id format is
   `{REGION_PREFIX}-025-{INDEX}` (for example, `AF-025-10000` for the
   Congo Basin reference tile).
3. Find the most recent `commitments/YYYY-WWW.json` file *before*
   the public-notification date.
4. Open the file. Look for the tile-id under the affected pathogen.
   - If present: the lead time is `notification_date - generated_at`.
     Risk score and confidence band are in the entry.
   - If absent: the commitment did not include this tile for that
     pathogen — record the miss.

The verification script at
[`scripts/verify_against_don.py`](../scripts/verify_against_don.py)
automates the lookup; pass a tile-id, pathogen, and notification
date, and it returns the relevant commitment entry (or none).

## Provenance

- Each commitment is generated by the live MCP at
  `https://qjtqgvpd9s.eu-west-1.awsapprunner.com/mcp` via a private
  GitHub Action that runs every Monday at 09:00 UTC.
- The action records the App Runner image digest at the time of
  generation in the file's `model_digest` field.
- The action commits the file to this repository with the founder's
  GPG-signed git author. GitHub's verified-commit badge is the
  on-platform proof of authorship and timestamp.
- Each new commitment file is also auto-snapshotted to the
  Internet Archive (`web.archive.org`) so the public timestamp does
  not depend on github.com remaining honest.

## Honest limits

- AqtaBio's seeded tile coverage is sparse at the operational tier
  (578 tiles vs an 80,000+ tile roadmap). Outbreaks in unseeded
  tiles cannot be evaluated against this ledger; they are recorded
  as `coverage_gap`, not `miss`.
- Five of eight priority pathogens (Ebola, H5N1, CCHF, West Nile,
  SARS-CoV-2) have live tile predictions; Mpox, Nipah, and
  Hantavirus are trained but pending tile seeding and are not on
  the weekly commitment.
- The risk score is population-level for a 25 km tile. It is not a
  per-patient diagnostic and not a forecast of when or by which
  pathogen an outbreak will occur.
- A high-risk commitment that is followed by no outbreak is not a
  false positive in the conventional ML sense — environmental
  conditions favouring spillover are necessary but not sufficient.
  The ledger surfaces these as `no_event_observed` for the
  evaluation window; they are the basis for the prevented
  fraction and counter-factual analyses.

## Why it matters

The aggregate validation across the full 25-event historical cohort
is the deliverable of the forthcoming medRxiv preprint (target Q3
2026). This ledger is the prospective complement: it builds, week
by week, a public record that no model architecture choice or
training-time decision can retroactively influence. The two
together — backtest plus prospective track record — are the
evidence base AqtaBio expects to be evaluated against.
